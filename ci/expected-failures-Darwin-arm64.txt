# ── Dated snapshot rigs (third-party/snapshots/, wide main-rig mirrors per era) ──
# These rigs resolve ~1900 crates each at point-in-time versions, so the FULL
# per-platform failure set is populated by the matrix sweep (run it via
# workflow_dispatch, then add the new failures here). Pre-seeded below are the
# confident, platform-class failures:
#
#  windows-sys 0.52/0.59 hit `raw-dylib` (E0455, Windows-only) on mac ONLY in the
#  2025-11/2026-05 rigs. Root cause: the fixups/windows-targets cfg is gated to
#  cfg(target_os="windows"), but reindeer can only emit a platform-gated cfg when
#  the crate already has a per-platform attr (deps) to anchor a platform={} map.
#  In these two rigs windows-targets-0.52 resolves WITHOUT its per-arch import-lib
#  deps, so reindeer flattens `--cfg=windows_raw_dylib` to UNCONDITIONAL → the
#  link! macro emits raw-dylib on mac. The 2024-11/2025-05 rigs (and the main rig)
#  keep the per-arch deps, so the cfg stays gated and they build fine on mac —
#  hence not listed (verified 2026-06-22). Real fix is a reindeer-fork change:
#  don't flatten a strict-subset cfg gate to unconditional. Until then, except:

#  catalogs any failure there.
